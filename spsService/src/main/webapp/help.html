URL Structure [base] is your location to access the rest API
For Example running the server on a local pc my [base] = http://localhost:8080/spsService/api/
For the one running on amazon [base] = http://23.23.181.154:8080/spsService/api
Resource URL METHOD Posted Data Query Parameters Description
Utility [base]/util/xmltojson POST any spsRequest data, the json representation will be returned None returns the request data in json format, good for if you have an xml request that you need to create json data for. THIS SHOULD BE USED FOR TESTING ONLY, nothing should actively use this api in production
User [base]/user/{userId} GET None
  • action - string - "create" to create, "login" to login
  • showGS - boolean - Show Game Sessions
  • showQuestions - boolean - Show Questions
  • showTransactions - boolean - Show Transactions
 Returns spsReply.userReply information for given userId
[base]/user POST spsRequest with userRequest node filled (see snippets) Will perform the associated action and return an spsReply.userReply
[base]/user/{userId} PUT spsRequest with userRequest node filled (see snippets) Will update the user based on given userId and return an spsReply.userReply
[base]/user/{userId}/gamesession/{gameSessionId}/question/{questionId}/answer/ POST spsRequest with answerRequest node filled (see snippets) spsReply.gameSessionReply filled with gameSession and answer information
[base]/user/{userId}/gamesession/{gameSessionId}/question/{questionId}/answer/{answerId} PUT spsRequest with answerRequest node filled (see snippets) Will update an answer based on given request and return spsReply.gameSessionReply filled with gameSession and answer information
[base]/user/{userId}/gameSession/{gameSessionId} GET None spsReply.userReply filled with gameSession and answer information
[base]/user/{userId}/gamesession/{gameSessionId} POST Will add a user to a gamesession
[base]/user/{userId}/transactions GET None Returns spsReply.userReply with shareTransaction node(s) filled for given userId
[base]/user/{userId}/transactions POST spsRequest with userRequest.shareTransaction node filled (see snippets) Will perform the associated action and return an spsReply.userReply
Transaction [base]/transaction/{transactionId} GET None None Returns spsReply.userReply information with shareTransaction node filled for given transactionId
Question [base]/question/{questionId} GET None None Returns spsReply.questionReply information for given questionId
[base]/question/ POST spsRequest with userRequest node filled (see snippets) None Will add a question and return an spsReply.questionReply
[base]/question/{questionId} PUT spsRequest with questionRequest node filled (see snippets) Will update the question based on given questionId and return an spsReply.questionReply
GameSession [base]/gamesession/{gameSessionId} GET None
  • showUsers = true/false - return all users in game session
  • showQuestions = true/false - return all questions for gameSession
 Returns spsReply.gameSessionReply information for given gameSessionId
[base]/gamesession/ POST spsRequest with gameSessionRequest node filled (see snippets) Will add a gameSession and return an spsReply.gameSessionReply
[base]/gamesession/{gameSessionId} PUT spsRequest with gameSessionRequest node filled (see snippets) Will update the gameSession based on given questionId and return an spsReply.gameSessionReply
[base]/gamesession/{gameSessionId}/question/ POST spsRequest with gameSessionRequest.gameSession.question node(s) filled (see snippets) Will add a question to a gameSession and return an spsReply.gameSessionReply. NOTE: If you want to add an existing question to a game session, make sure you pass in the 'id' node inside of the 'question' node
